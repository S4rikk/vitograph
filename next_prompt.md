# 📝 Техническое задание: Жёсткая фиксация высоты окна чата на Десктопе (Изолированный скролл)

Привет! Из-за использования lex-1 и min-h в последних правках у нас "сломалась" десктопная версия. Окно чата перестало иметь жесткую границу высоты и теперь разворачивается вниз "бесконечной портянкой", заставляя пользователя скроллить всю страницу браузера, чтобы добраться до поля ввода.
Задача — вернуть десктопной версии (от брейкпоинта sm:) строгую высоту, при которой чат имеет свой внутренний аккуратный скролл-бар, а поле ввода всегда прибито к низу окна. При этом мобильная версия должна остаться такой же, как сейчас (тянущейся).

**ИСПОЛЬЗУЕМЫЕ СКИЛЛЫ ДЛЯ ЭТОЙ ЗАДАЧИ:** react-ui-patterns, tailwind-responsive.

### Шаг 1: Фиксация высоты на Десктопе
Открой pps/web/src/components/assistant/AiAssistantView.tsx.
Найди главный оберточный <div className="..."> (примерно строка 354), внутри которого лежит шапка и HealthGoalsWidget.

Сейчас его классы:
lex flex-1 flex-col overflow-hidden rounded-xl border border-cloud-dark bg-white shadow-sm sm:min-h-[600px] mb-[env(safe-area-inset-bottom)]

- На десктопе убираем поведение "тянучки" (lex-1) и жестко фиксируем высоту: добавь классы sm:h-[650px] sm:flex-none.
- Также удали старый класс sm:min-h-[600px].

Итоговый список классов должен выглядеть примерно так:
lex flex-1 sm:flex-none flex-col overflow-hidden rounded-xl border border-cloud-dark bg-white shadow-sm sm:h-[650px] mb-[env(safe-area-inset-bottom)]

### Шаг 2: Проверка
На мобилке (< sm) чат останется свободным (работает lex-1), а начиная с десктопа он будет представлять собой жесткий блок 650 пикселей высотой, внутри которого сообщения будут скроллиться (так как контейнер сообщений имеет lex-1 overflow-y-auto).

Сделай локальный коммит. Деплой не нужен.
