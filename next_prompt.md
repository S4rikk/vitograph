# TASK: Реализация модуля Распознавания Этикеток (Label Scanner)

## Общее Описание
Необходимо создать новую full-stack фичу для сканирования этикеток и составов продуктов (OCR + AI Анализ). Фича включает расшифровку E-добавок и вынесение краткого, конкретного вердикта (Красная, Желтая или Зеленая зона) в зависимости от контекста здоровья и целей пользователя.

Архитектурный концепт: `C:\project\kOSI\docs\arch_label_scanner.md`

## Требования к реализации

### 1. Frontend (`FoodInputForm.tsx`)
- Добавить новую кнопку-иконку "🏷️ Этикетка" (или иконку штрихкода/лупы) **строго между** кнопкой "Фото" и полем "Вес (г)" во втором ряду формы.
- Кнопка должна открывать камеру/выбор фото аналогично текущей кнопке "Фото".
- При получении результата показать UI-плашку ("Светофор") над полем ввода:
  - 🔴 Красная зона (bg-red-50, border-red-200)
  - 🟡 Желтая зона (bg-yellow-50, border-yellow-200)
  - 🟢 Зеленая зона (bg-green-50, border-green-200)
- В плашке вывести:
  - Вердикт (краткое и четкое пояснение).
  - Списком найденные E-коды с расшифровкой и индексом вредности.
- Автоматически подставить распознанное `product_name` в поле "Блюдо", чтобы юзер мог сразу ввести вес и сохранить.

### 2. API Client (`api-client.ts`)
- Добавить метод `analyzeLabel(base64Image: string): Promise<LabelScannerOutput>` по аналогии с `analyzeFood`.

### 3. Backend Routes & Controller (`ai.routes.ts`, `ai.controller.ts`)
- Создать POST-маршрут `/vision/label`.
- В контроллере:
  1. Вызвать `fetchUserContext()` (но для экономии токенов извлечь только аллергии, текущие ограничения по здоровью и цели диеты).
  2. Передать урезанный профиль и Base64-картинку в новый сервис `analyzeLabelImage`.

### 4. AI Logic (`label-scanner.ts` - новый файл в `apps/api/src/ai/src/graph/` или `src/llm/`)
- Использовать `callLlmStructured` с моделью GPT-4o.
- Схема (`LabelScannerOutputSchema`):
```typescript
z.object({
  product_name: z.string(),
  verdict: z.enum(["GREEN", "YELLOW", "RED"]),
  verdict_reason: z.string().describe("Краткое, четкое, конкретное пояснение полезности данного продукта для ЭТОГО пользователя"),
  e_codes: z.array(z.object({
    code: z.string(),
    name: z.string(),
    danger_level: z.enum(["LOW", "MEDIUM", "HIGH"]),
    description: z.string()
  })).optional()
})
```
- Системный промпт должен **строго** требовать проверку состава по предоставленному профилю здоровья пользователя (аллергии, заболевания, диета).
