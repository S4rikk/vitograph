# ⚠️ КОРРЕКТИРОВКА К ПЛАНУ: Частичный запрет на `<nutr>` теги

Парсер `food-log-parser.ts` использует теги `<nutr>` для извлечения **микронутриентов** (Железо, Кальций, Калий и т.д.) и отображения их в карточке `FoodCard`. Если ИИ перестанет генерировать эти теги, секция «МИКРОНУТРИЕНТЫ» в карточке станет пустой.

## Правильная логика при записи еды

Ответ ИИ при записи еды должен состоять из **трёх частей**:

### 1. Техническая строка (без изменений)
```
Записал 350г гречка с печенью: 450 ккал, 30г белков, 15г жиров, 55г углеводов
```

### 2. Теги — ОБЯЗАТЕЛЬНЫ (парсятся фронтендом для карточки)
```
<meal_score score="75" reason="Сбалансированный обед" />
<nutr type="micro">Железо 5мг</nutr><nutr type="micro">Цинк 3мг</nutr>
```

### 3. Текстовый ответ — тут действует запрет
Описательный стиль, без цифр, без списков через запятую, без тегов `<nutr>`:
```
Отличный выбор на обед! Гречка с печенью — питательное блюдо, богатое железом и цинком. Хороший выбор для энергии!
```

## Резюме
Запрет на `<nutr>` касается **ТОЛЬКО текстовой части** ответа. Технические теги для парсера карточки — обязательны.
