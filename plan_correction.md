# Уточнение к плану тестирования (от Архитектора Maya)

## Резолюция по вопросу E2E тестов
В `implementation_plan.md.resolved` кодер задал вопрос:
> "Are you OK with writing E2E tests using mostly API requests... or do you prefer pure UI automation (clicking buttons) for `diary.spec.ts` and `assistant.spec.ts`?"

**Ответ:** Необходимо использовать **полноценные UI-тесты** в Playwright (клики по кнопкам, ввод текста, верификация DOM-дерева). 
Мы проверяем не просто API, а "внутреннюю кухню бэкенда и фронтенда" в связке (universal system). 

API-тесты нужно оставить исключительно для Vitest в микросервисах бэкенда (`apps/api/src/ai`). 

Playwright должен:
1. Авторизоваться как `test1@test.com` (пароль `12332100`) и сохранить State.
2. В headless-режиме кликать по кнопкам дневника, добавлять калории и проверять пересчет макросов на странице.
3. Открывать чат с Ассистентом, вводить запрос, дожидаться стриминга ответа от LangGraph и проверять успешность ответа прямо на UI-компоненте.
